# 智能分析系统实施规划（Phase 详细执行版）

## 0. 规划目标

把“智能分析系统”落成可演示 MVP，覆盖：
1. 前后端基础框架可运行、可测试；
2. 前端 UI 完整可用；
3. 后端真实接口可提供 NL2SQL 能力；
4. 前后端稳定联调并可验收。

---

## 1. 交付边界（MVP）

### 1.1 必做范围
- 会话管理：创建、切换、重命名、删除、历史回放。
- 智能问答：自然语言 -> SQL（只读）-> 查询结果 -> 文本结论。
- 图表展示：柱状图/折线图/饼图 + 表格回退。
- 记忆追问：同会话上下文有效。
- 异常处理：超时、无数据、非法 SQL、服务错误可读提示。

### 1.2 非本期范围
- 多数据库（MySQL/PostgreSQL）
- 权限系统/多租户
- 报表编排与导出中心

---

## 2. 技术实现基线

### 前端
- React + TypeScript + Vite
- Zustand（本地状态）+ React Query（服务端状态）
- ECharts 图表

### 后端
- FastAPI + Uvicorn
- LangChain + Qwen3
- SQLite（业务数据 + 会话数据）

### 关键目录建议
- `backend/app/main.py`
- `backend/app/api/`
- `backend/app/services/`
- `backend/tests/`
- `frontend/src/components/`
- `frontend/src/layout/`
- `frontend/src/services/api.ts`
- `frontend/src/store/`

---

## 3. Phase 详细拆分

## Phase 1：基础框架搭建 + 运行测试（D1-D2）

### 开发任务
- 后端脚手架、配置、日志、异常中间件、`/health`、`/version`。
- 前端脚手架、路由、API 客户端、全局错误提示。
- 前后端本地联通机制（前端启动时健康检查）。

### 测试任务
- 后端：`pytest` 至少覆盖健康检查、异常处理。
- 前端：`vitest` 至少覆盖 App 渲染、API 层调用。

### 交付物
- 可运行的前后端项目。
- 基础测试可通过。

### 退出标准（DoD）
- 一键启动可用。
- 测试命令可执行且通过。
- 基础结构支持后续迭代。

---

## Phase 2：前端 UI 研发（D3-D5）

### 开发任务
- 三栏布局：会话、聊天、图表。
- SessionPanel：CRUD 交互与空态。
- ChatPanel：消息流、输入发送、失败重试、滚动控制。
- ChartPanel：ECharts + fallback table + 空态占位。
- 接入 Zustand + React Query。

### 测试任务
- 组件交互测试。
- 页面主流程测试（mock API）。

### 交付物
- 可演示的前端 UI。
- mock 数据下流程完整可走通。

### 退出标准（DoD）
- UI 达到演示可用标准。
- 关键交互无阻塞性问题。

---

## Phase 3：后端接口研发（D6-D9）

### 开发任务
- 会话 API：
  - `POST /api/sessions`
  - `GET /api/sessions`
  - `PATCH /api/sessions/{session_id}`
  - `DELETE /api/sessions/{session_id}`
  - `GET /api/sessions/{session_id}/messages`
- 聊天 API：
  - `POST /api/chat`
  - 接入 Qwen3 + LangChain（NL2SQL）
  - SQLite 执行器与结果结构化
  - 输出 `answerText/tableData/chartSpec/trace`
- 记忆策略：recent window + summary。

### 安全稳定任务
- SQL 只读校验（禁 DML/DDL）。
- 表白名单、行数限制、超时机制。
- 统一错误码和可读错误消息。

### 测试任务
- 单测：SQL 拦截、chart 映射、memory 组装。
- 接口测：成功、无数据、非法 SQL、超时、异常兜底。

### 交付物
- 可被前端直接调用的真实 API。
- 接口测试报告。

### 退出标准（DoD）
- 核心接口稳定可用。
- 异常路径可控且可追踪。

---

## Phase 4：前后端联调（D10-D12）

### 开发任务
- 字段契约对齐：命名、空值、时间格式、错误码。
- 打通主流程：创建会话 -> 问答 -> 图表展示 -> 追问。
- 缺陷池管理：P0/P1/P2 分级修复。

### 测试任务
- 冒烟测试：主流程全通过。
- 回归测试：修复后关键流程不回退。
- 性能基线：端到端耗时与慢链路识别。

### 交付物
- 联调缺陷清单与关闭记录。
- 联调通过报告。

### 退出标准（DoD）
- 无阻塞性联调缺陷。
- 可稳定演示完整业务链路。

---

## Phase 5：收口与发布准备（D13-D14）

### 开发任务
- 完善部署文档、环境变量模板、启动脚本。
- 固化可观测字段：`request_id/session_id/sql/latency/error_code`。
- 完成最终 UAT、演示脚本、交付包整理。

### 交付物
- 发布文档与运行手册。
- 最终验收记录。

### 退出标准（DoD）
- MVP 范围冻结。
- 项目达到可试运行状态。

## 4. API 契约（联调基线）

### 4.1 Chat 请求
```json
{
  "sessionId": "s_123",
  "message": "近30天各区域销售额趋势"
}
```

### 4.2 Chat 响应
```json
{
  "answerText": "华东区域整体上升，华北近一周有回落。",
  "tableData": [{"region": "华东", "date": "2026-04-01", "sales": 12345}],
  "chartSpec": {"type": "line", "xField": "date", "yField": "sales", "seriesField": "region"},
  "trace": {"sql": "SELECT ...", "latencyMs": 640, "requestId": "r_abc"}
}
```

## 5. 验收清单（最终）

- [ ] 前后端可独立启动且可连通。
- [ ] 会话管理全流程可用。
- [ ] chat 接口可返回真实查询结果。
- [ ] 图表可按数据自动映射，失败可回退表格。
- [ ] 支持同会话上下文追问。
- [ ] 异常路径提示清晰，日志可追踪。
- [ ] 联调通过，演示脚本稳定执行。

---

## 6. 风险与预案

1. NL2SQL 准确率波动
   - 预案：补充 schema 描述 + few-shot + SQL 后校验。
2. 图表自动映射不稳定
   - 预案：优先限定 3 类图表，无法映射统一回退表格。
3. 响应延迟过高
   - 预案：限制结果规模、优化提示词、缓存高频问题。
4. 联调反复返工
   - 预案：提前冻结 API 契约并每日缺陷评审。

---
