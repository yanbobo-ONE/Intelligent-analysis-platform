# 前后端接口对齐计划

## 目标

统一前后端接口契约，减少联调阶段的字段不一致、流式协议不一致、错误码不一致问题，确保智能分析系统能够稳定支撑会话、聊天、NL2SQL、图表渲染与消息历史能力。

## 对齐范围

### 1. 会话接口
- `POST /api/sessions`
- `GET /api/sessions`
- `PATCH /api/sessions/{session_id}`
- `DELETE /api/sessions/{session_id}`
- `GET /api/sessions/{session_id}/messages`

### 2. 聊天接口
- `POST /api/chat`
- `POST /api/chat/stream`

### 3. 统一返回结构
- `answer_text`
- `table_data`
- `chart_spec`
- `trace`

### 4. 统一 trace
- `model`
- `latency_ms`
- `sql`
- `tool_calls`
- `streaming`

### 5. 错误体系
- 空输入
- 会话不存在
- 非法 SQL
- 模型调用异常
- 流式输出异常
- 依赖缺失

## 前端数据模型

### SessionItem
- `id`
- `title`
- `status`
- `created_at?`
- `updated_at?`

### MessageItem
- `id`
- `role`
- `content`
- `created_at?`

### BackendSession
- `id`
- `title`
- `created_at`
- `updated_at`

### BackendMessage
- `id`
- `session_id`
- `role`
- `content`
- `created_at`

## 后端返回示例

### /api/chat
```json
{
  "answer_text": "华南区域销售额最高，其次是华东和华北。",
  "table_data": [["华南", 1560.2], ["华东", 1200.5], ["华北", 980.0]],
  "chart_spec": {
    "type": "bar",
    "xField": "region",
    "yField": "total_amount",
    "seriesField": "region"
  },
  "trace": {
    "model": "qwen3-max",
    "latency_ms": 1454,
    "sql": "SELECT region, SUM(amount) AS total_amount ...",
    "tool_calls": [],
    "streaming": false
  }
}
```

### /api/chat/stream
```text
data: ...

data: [DONE]
```

## 验收标准
- 前端能完整读取会话列表和消息历史。
- 前端能稳定发送聊天消息并展示流式输出。
- 后端能稳定返回 NL2SQL 标准结构。
- 任何错误都能返回可读且可追踪的错误信息。
- 前后端字段命名完全一致。

## 后续建议
- 将该计划作为联调阶段的唯一契约基线。
- 在接口变更前先更新该计划，再同步实现代码与测试。
